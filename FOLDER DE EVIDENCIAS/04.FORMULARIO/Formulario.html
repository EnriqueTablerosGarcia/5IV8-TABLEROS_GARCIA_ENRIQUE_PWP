Todos los formularios en la web, manejan metodos para el envio de las diferentes peticiones que realiza el usuario. Considerando la arquitectura Cliente-servidor, el manejo de las peticiones se hace por medio del protocolo HTTP y actualmente por x HTTPS, cada peticion debe de ser atendida en un tiempo de respuesta conforme al modelo TCP/IP. Los metodos principales de los formularios son:

  • GET

    Este método se encarga de enviar la información del formulario por medio de la salida estándar (conocida como url) del navegador, esto propicia que puedan existir inyecciones de código realizando violaciones a la seguridad del sistema.

  • POST

    Este método se encarga de enviar la información para poder registrar un cambio o un nuevo registro dentro del sistema, enviándola igual por el método estándar; pero mediante un ocultamiento, esto significa que no es visible para el usuario. Lo que nos ofrece un mayor grado de seguridad en el manejo de información.

En resumen los metodos son:

  • GET

    Sirve para enviar y realizar operaciones de lectura o consulta

  • POST

    Sirve para enviar información de un recurso al servidor, con el fin de registrarlo

  • DELETE

    Sirve para eliminar un recurso específico dentro del servidor obteniendo una respuesta de actualización

  • PUT

    Sirve para almacenar la información de un recurso particular y conectarse al servicio para realizar la actualización

  • HEAD

    Sirve para regresar el código de puesta de cualquier cabecera HTTP asociada a un recurso
