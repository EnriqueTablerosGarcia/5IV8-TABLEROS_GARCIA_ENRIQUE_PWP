PRÁCTICA 17 - API CRUD COMPLETA

Sistema Full Stack con Backend y Frontend Separados

Descripción del Proyecto

API CRUD completa con arquitectura separada entre Backend y Frontend, implementando todas las operaciones de gestión de productos con categorías.

  • Backend: API REST con Node.js, Express y MySQL
  • Frontend: Interfaz de usuario con HTML, CSS y JavaScript
  • Separación de Responsabilidades: Backend y Frontend completamente independientes
  • Gestión de Productos: CRUD completo con categorías
  • Validaciones: Tanto del lado del cliente como del servidor

Arquitectura del Sistema

Backend

  • Server.js - Servidor Express
  • Controllers - Lógica de negocio
  • Models - Modelos de datos
  • Routes - Definición de endpoints
  • Config - Configuración de BD

Frontend

  • Public - Archivos estáticos

Endpoints de la API

  • GET /products - Obtener todos los productos
  • GET /products/:id - Obtener un producto por ID
  • POST /products - Crear un nuevo producto
  • PUT /products/:id - Actualizar un producto
  • DELETE /products/:id - Eliminar un producto
  • GET /categories - Obtener todas las categorías
