PRÁCTICA 16 - CRUD SEPARADO CON MVC

Arquitectura Modelo-Vista-Controlador con Express.js

Descripción del Proyecto

Sistema CRUD implementado con arquitectura MVC (Modelo-Vista-Controlador) que separa la lógica de negocio, la presentación y el control de flujo en diferentes módulos.

  • Modelo: Gestión de datos y lógica de base de datos
  • Vista: Presentación de datos con EJS
  • Controlador: Lógica de negocio y manejo de peticiones
  • Rutas: Definición de endpoints de la API
  • Base de Datos: MySQL para gestión de cursos

Tecnologías Utilizadas

Node.js
Express.js
MySQL2
EJS
dotenv
CORS
Body-Parser

Estructura del Proyecto MVC

16.CRUDseprado/ ├── Controllers/ │ └── cursosControl.js # Controladores de cursos ├── database/ │ └── db.js # Configuración de BD ├── routers/ │ └── cursosRouters.js # Rutas de la API ├── views/ │ ├── bienvenido.html # Página de inicio │ └── cursos.ejs # Vista de cursos ├── index.js # Servidor principal └── package.json # Dependencias

Características Principales

Separación de Responsabilidades

Código organizado en capas independientes siguiendo el patrón MVC

CRUD Completo

Operaciones Create, Read, Update y Delete para cursos

API RESTful

Endpoints bien definidos siguiendo convenciones REST

Base de Datos MySQL

Gestión persistente de información de cursos

Vistas Dinámicas

Renderizado del lado del servidor con EJS

Middleware Personalizado

CORS, JSON parser y manejo de errores

Endpoints de la API

  • GET /cursos - Obtener todos los cursos
  • GET /cursos/:id - Obtener un curso por ID
  • POST /cursos/registrar-curso - Crear un nuevo curso
  • PUT /cursos/:id - Actualizar un curso existente
  • DELETE /cursos/:id - Eliminar un curso
  • GET /vista/cursos-ejs - Vista de cursos con EJS

Configuración de Base de Datos

El proyecto utiliza MySQL para almacenar la información de cursos. La conexión se gestiona a través del módulo mysql2 con las siguientes características:

  • Host: localhost
  • Base de datos: cursosdb
  • Tabla principal: cursos
  • Pool de conexiones para mejor rendimiento
